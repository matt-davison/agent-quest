# ASCII Visualizer Reference

This guide provides detailed patterns and procedures for generating evocative ASCII art during Agent Quest gameplay.

## Philosophy

ASCII art in Agent Quest serves to:
1. **Immerse** players in the world
2. **Communicate** tactical information clearly
3. **Reward** exploration with visual discovery
4. **Integrate** with the game's digital-fantasy aesthetic

The art should feel like output from an ancient terminal discovering a magical world - clean geometry mixed with mystical symbols.

1. **Determine size** from battle-environment terrain.size
2. **Draw border** (walls or open edges)
3. **Place terrain features** (cover, elevation, difficult terrain)
4. **Place hazards** with warning symbols
5. **Place interactive elements** marked with `*`
6. **Place combatants** (@ for player, E/â˜  for enemies)
7. **Add legend** explaining symbols
8. **List combatant stats** below map

## Integration with Game Loop

### When to Generate Art

| Trigger | Art Type | Scope |
|---------|----------|-------|
| **LOOK** | Full panorama | Entire location |
| **Enter area** | Area scene | Specific area |
| **Combat starts** | Battle map | Combat zone |
| **Explore dungeon** | Dungeon map | Discovered rooms |
| **Major discovery** | Special scene | Key moment |

### Updating Battle Maps

During combat, update the map to show:
- Position changes
- Destroyed cover
- Triggered hazards
- Defeated enemies (replace with `X`)
- Health changes (in legend)

### Session Continuity

For returning players, you can reference previous scenes:
- "The library looks just as you left it..."
- Show persistent changes (destroyed areas, unlocked doors)
